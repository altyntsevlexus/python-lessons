Сьогодні навчимося писати цикл for — одну з найпотужніших ідей у програмуванні. Замість восьми однакових рядків — три. Читай кожен крок по порядку і запускай приклади в Trinket.

Самостійне вивчення Python · Turtle 7 клас ~45 хвилин
00
Проблема — занадто багато рядків
Зв'язок з уроком 1 · навіщо потрібен цикл

На уроці 1 ти малював квадрат так: написав forward + right чотири рази вручну. Це 8 рядків коду для однієї простої фігури.

А тепер уяви: що якби треба було намалювати 36-кутник або 100-кутник? Це 72 або 200 рядків одноманітного коду!

Квадрат БЕЗ циклу — 8 рядків
# Повторюємо вручну 4 рази t.forward(100); t.left(90) t.forward(100); t.left(90) t.forward(100); t.left(90) t.forward(100); t.left(90)
Квадрат З циклом — 3 рядки ✨
# Цикл робить це сам! for i in range(4): t.forward(100) t.left(90)

Це не просто менше рядків — це інший спосіб думати. Цикл каже: «повтори це 4 рази», і комп'ютер сам рахує.

01
Цикл for — як записати
синтаксис · двокрапка · відступ

Цикл for складається з двох частин:

for i in range(4):   # ← заголовок: двокрапка обов'язкова!
    t.forward(100)  # ← тіло: Tab (= 2 або 4 пробіли)
    t.left(90)     # ← тіло: теж з відступом
ЧастинаЩо означає
for i in range(4): Повторити тіло 4 рази; змінна i по черзі приймає значення 0, 1, 2, 3
Двокрапка : Обов'язковий символ після заголовку — без неї SyntaxError
Відступ (1 Tab) Все, що є частиною тіла циклу, повинно мати відступ. Один Tab = 2 або 4 пробіли. Trinket додає його автоматично після двокрапки
i Лічильник — змінна, яка зростає на кожній ітерації. Назву можна змінити: step, x, n — але i — традиція

Що таке відступ (Tab) і чому Python без нього не працює?

Коли ти натискаєш клавішу Tab на клавіатурі, курсор зміщується вправо на 4 пробіли. Саме так Python визначає, які рядки належать всередині циклу, а які виконуються після нього.

На відміну від багатьох інших мов, Python не використовує фігурні дужки {} для позначення блоків коду. Замість них — відступи. Це не просто стиль оформлення: це вимога мови.

✓ Правильно — з відступом (Tab)
for i in range(4): t.forward(100) # ← Tab! t.left(90) # ← Tab! # без відступу — вже поза циклом
✗ Помилка — без відступу
for i in range(4): t.forward(100) # ← без Tab! t.left(90) # → IndentationError ↑
Добра новина: Trinket.io автоматично додає Tab після двокрапки — просто натисни Enter і редактор сам зробить відступ. Якщо випадково видалив — натисни Tab на початку рядка. Усі рядки всередині одного циклу повинні мати однаковий відступ.
Типові помилки — прочитай до практики!
  • Забута двокрапка після range(n) → SyntaxError
  • Немає відступу в тілі → IndentationError
  • range(4) → 0, 1, 2, 3 — чотири числа, але починає з нуля! Не 1, 2, 3, 4
  • Змішані пробіли і Tab → TabError. Використовуй тільки Tab або тільки пробіли
Спробуй сам!
  • Напиши квадрат через цикл і запусти — переконайся, що він малюється
  • Навмисне видали двокрапку → прочитай повідомлення помилки
  • Навмисне видали відступ → прочитай повідомлення помилки
02
Правильний n-кутник
формула кута · 360 ÷ n

Будь-який правильний n-кутник малюється за однією формулою: forward(сторона) + поворот на 360 ÷ n градусів, повторити n разів.

3
Трикутник
360 ÷ 3 = 120°
4
Квадрат
360 ÷ 4 = 90°
5
П'ятикутник
360 ÷ 5 = 72°
6
Шестикутник
360 ÷ 6 = 60°
36
36-кутник
360 ÷ 36 = 10°
∞
Майже коло
кут → 0°

Замість жорсткого числа — змінна n. Зміниш одне число — отримаєш іншу фігуру:

import turtle
t = turtle.Turtle()
t.speed(0)

n = 6  # ← змінити: 3, 4, 5, 6, 36, 72, 360 …
for i in range(n):
    t.forward(80)
    t.left(360 / n)

turtle.done()
n = 6 — шестикутник
n = 36 — майже коло

  • Запусти код з n=6 — шестикутник
  • Зміни n на 3, 5, 8 — порівняй результати
  • Поспробуй n=36, n=72 — що нагадує фігура?
  • Додай заливку fillcolor до свого n-кутника
03
for зі списком значень
for side in [...] · коли сторони різні

range(4) дає одне й те саме число чотири рази — підходить для квадрата. Але прямокутник має дві різні довжини сторін. Тут використовують список:

import turtle
t = turtle.Turtle()

# range(4) НЕ підходить — сторони різні
# for side in [120, 60, 120, 60] — кожне значення окремо
for side in [120, 60, 120, 60]:
    t.forward(side)
    t.left(90)

turtle.done()

На кожній ітерації side приймає наступне значення зі списку: спочатку 120, потім 60, потім знову 120, потім 60.

ЗаписКоли використовувати
for i in range(n): Коли всі сторони однакові і потрібно просто повторити n разів
for side in [a, b, c, d]: Коли сторони різні і треба задати кожну окремо
Спробуй сам!
  • Намалюй прямокутник 150×80 через список
  • Зміни числа у списку — отримай вузький і довгий прямокутник
  • Спробуй список з трьох різних чисел — яка фігура вийде?
04
range() — три форми
range(n) · range(a, b) · range(a, b, step)

range() — це генератор послідовності чисел. Він існує у трьох формах:

ФормаПослідовністьПриклад
range(n) 0, 1, 2, … n-1 range(4) → 0, 1, 2, 3
range(a, b) a, a+1, … b-1 range(1, 5) → 1, 2, 3, 4
range(a, b, step) a, a+step, a+2·step, … (до b не включно) range(0, 10, 2) → 0, 2, 4, 6, 8
range(a, b, step) від'ємний Можна рахувати назад range(10, 0, -2) → 10, 8, 6, 4, 2

Приклад: намалювати 5 кіл рівномірно по горизонталі від −200 до 200:

import turtle
t = turtle.Turtle()
t.speed(0)

# x = -200, -100, 0, 100, 200
for x in range(-200, 201, 100):
    t.penup()
    t.goto(x, 0)
    t.pendown()
    t.circle(30)

turtle.done()
Чому range(-200, 201, 100), а не range(5)?
Обидва варіанти дають однаковий результат. Але range(-200, 201, 100) одразу видно які координати використовуємо. З range(5) + формула треба думати: «0×100−200=−200, 1×100−200=−100…». Якщо мислиш координатами — пиши range(a, b, step).
Спробуй сам!

  • Зміни крок на 50 — скільки кіл стане?
  • Намалюй 4 квадрати 40×40 у рядок, починаючи від x=−100, крок 60
05
range(5) або range(-200, 201, 100)?
два записи — один результат · коли що обирати

На попередньому кроці ти малював 5 кіл через range(-200, 201, 100) і goto(). Той самий результат можна отримати через range(5) — просто переміщуючи черепашку вперед між колами.

range(-200, 201, 100) — через goto()
for x in range(-200, 201, 100): t.penup() t.goto(x, 0) t.pendown() t.circle(30)
range(5) — просто йдемо вперед
for i in range(5): t.circle(30) t.up() t.forward(50) # крок між колами t.down()
Коли що обирати?
range(a, b, step) + goto() — якщо важлива точна позиція: «хочу кола рівно в −200, −100, 0…».
range(n) + forward() — якщо важлива лише кількість, а розміщення відносне: «намалюй 5 кіл поряд».
Спробуй сам!

  • У варіанті з range(5) збільш крок з 50 до 80 — що змінилось?
  • Намалюй 8 кіл у рядок через range(8)
06
Лічильник i всередині циклу
i зростає · forward(i * 3) · спіраль

Змінна i — це не просто лічильник. Її можна використовувати всередині тіла циклу. На кожній ітерації i зростає, тому й результат змінюється.

import turtle
t = turtle.Turtle()
t.speed(0)

# i = 0, 1, 2, … 59
# forward(i * 3) = 0, 3, 6, 9, … 177
for i in range(60):
    t.forward(i * 3)   # крок зростає з i!
    t.left(89)

turtle.done()

Оскільки крок forward зростає з кожною ітерацією — черепашка малює спіраль. Кут визначає її форму:

КутРезультат
left(90)Квадратна спіраль
left(89) або left(91)Кругла розкручена спіраль
left(121)Трикутна спіраль
left(137)Золотий кут — як насіння соняшника
Що таке «золотий кут» 137°?
Це кут, який природа використовує для розташування насіння у соняшнику, пелюсток у квітах, листків на стеблі. Черепашка з left(137) і range(200) намалює точнісінько такий самий візерунок!
Спробуй сам!
  • Запусти код з left(89) — кругла спіраль
  • Зміни кут на 90, 91, 121, 137 (у left()) — порівняй форми
  • Збільш range до 200 і зміни множник: i * 2, i * 5
  • Додай t.pencolor для красивого кольору

Практика — Піаніно
самостійна робота · використай всі інструменти уроку

Це фінальне завдання уроку. Напиши код самостійно — жодного готового прикладу. Використай все, що вивчив сьогодні.

Завдання: намалювати клавіатуру піаніно — 8 білих клавіш і 7 чорних клавіш між ними.

Підказка — план дій:
Розбий задачу на дві частини і вирішуй по черзі. Спочатку намалюй тільки білі клавіші — переконайся, що вони правильно розміщені. Потім додай чорні.

Білі клавіші — що потрібно знати:

  • 8 клавіш однакового розміру: ширина 28, висота 100
  • Відстань між лівими краями сусідніх клавіш — 30 пікселів
  • Використай range(a, b, step) для перебору x-позицій
  • Кожна клавіша — прямокутник: намалюй його циклом через список сторін [28, 100, 28, 100]
  • Не забудь penup() → goto() → pendown() перед кожною клавішею
  • Заливка білим: fillcolor("white") + begin_fill() / end_fill()

Чорні клавіші — що потрібно знати:

  • 7 чорних клавіш, розміром 16×60
  • Стартові x-позиції — ті самі, що у білих (але тільки перші 7), зміщені вправо на 20 пікселів
  • По y підніми вгору так, щоб чорна клавіша виступала зверху над білою
  • Заливка чорним: fillcolor("black")
