IF / ELIF + RANDOM

Сьогодні програма навчиться думати: if/elif/else дозволяють їй робити вибір залежно від умов, а random додає справжній елемент сюрпризу. Кожен запуск — новий результат.

00
Проблема — програма завжди робить одне й те саме
Зв'язок з уроком 2 · навіщо потрібне розгалуження

На уроці 2 ти навчився малювати фігури через for. Але при кожному запуску результат однаковий — програма «сліпа». Що якби вона могла дивитись на дані і вирішувати що малювати?

Без if — завжди однаково
# Завжди малює коло одним кольором t.pencolor("gold") t.circle(20) # Не важливо, який розмір — # колір завжди золотий
З if — вибір залежно від умови
# Колір залежить від розміру! r = 20 if r > 15: t.pencolor("gold") # велике → золоте else: t.pencolor("white") # мале → біле t.circle(r)

Саме це робить програму «розумною». Вона дивиться на дані і сама приймає рішення.

01
if / else — як записати
синтаксис · двокрапка · відступ · ті самі правила що й у for

Конструкція if/else має дві гілки: якщо умова виконується — перша, інакше — друга:

if умова:            # ← двокрапка обов'язкова!
    # тіло — виконується якщо ПРАВДА
    t.pencolor("red")
else:                # ← теж двокрапка
    # тіло — виконується якщо НЕПРАВДА
    t.pencolor("blue")
ЧастинаЩо означає
if умова: Перевіряє умову. Якщо вона True — виконується тіло if
else: Виконується, якщо умова False. Не обов'язкова — можна мати тільки if
Двокрапка : Обов'язкова після if умова і після else — без неї SyntaxError
Відступ (Tab) Тіло кожної гілки повинно мати відступ — ті самі правила, що й у for 
import turtle
t = turtle.Turtle()
t.speed(0)

r = 25   # ← спробуй змінити на 5 або на 30

if r > 15:
    t.pencolor("gold")
    t.pensize(3)
else:
    t.pencolor("white")
    t.pensize(1)

t.circle(r)
turtle.done()
Типові помилки — прочитай до практики!
  • Забута двокрапка після if умова або після else → SyntaxError
  • Немає відступу в тілі → IndentationError (ті самі правила, що й у for!)
  • else не буває без if — тільки після нього
  • Після else — тільки двокрапка, ніякої умови!
Спробуй сам!
  • Запусти з r = 25 — яке коло? Зміни на r = 5 — що змінилось?
  • Навмисне видали двокрапку після else → прочитай повідомлення про помилку
  • Навмисне видали відступ у тілі → прочитай повідомлення
02
if / elif / else — ланцюжок умов
elif · кілька гілок · спрацьовує перша підходяща

Між if і else можна додати скільки завгодно elif («else if» — «інакше якщо»). Намалюємо коло — колір залежить від розміру, три варіанти:

import turtle
t = turtle.Turtle()
t.speed(0)

r = 15   # ← спробуй: 5, 15, 25

if r > 20:                     # якщо r більше 20
    t.pencolor("gold")       # → золотий
elif r > 10:                  # якщо 1-а не спрацювала, але r > 10
    t.pencolor("#5b9cf6")   # → синій
else:                          # якщо жодна умова не підійшла
    t.pencolor("white")     # → білий

t.circle(r)
turtle.done()
Ключове правило: Python перевіряє умови зверху вниз і виконує тільки першу гілку, яка підійшла. Решта — ігноруються. Якщо r = 25, спрацює gold і більше нічого не перевіряється.
rЯка умова спрацьовує?Колір кола
r = 25r > 20 → True → зупиняємосьgold
r = 15r > 20 → False; r > 10 → True → зупиняємось#5b9cf6
r = 5r > 20 → False; r > 10 → False → elsewhite
Спробуй сам!
  • Запусти з r = 5, r = 15, r = 25 — чи бачиш усі три кольори?
  • Додай ще один elif — наприклад elif r > 5: → зелений
  • Перевір: чи може одночасно виконатись і if, і elif? (відповідь: ні)
03
Оператори порівняння
==, !=, >, <, >=, <= · True і False · = не == !

Умова в if завжди дає або True (правда), або False (неправда). Для запису умов — шість операторів:

ОператорОзначенняПрикладРезультат
==Рівно (порівняння)5 == 5True
!=Не рівно5 != 3True
>Більше10 > 5True
<Менше3 < 7True
>=Більше або рівно5 >= 5True
<=Менше або рівно3 <= 3True
Найчастіша помилка — = замість == !

= — це присвоєння (дати змінній значення): r = 10
== — це порівняння (перевірити рівність): if r == 10:

Якщо написати if r = 10: → SyntaxError. Python суворо розрізняє ці операції.
Спробуй сам!
  • Напиши if i == 0: в циклі — що друкується тільки на першій ітерації?
  • Як перевірити, чи число не дорівнює нулю? (підказка: !=)
  • Навмисне напиши if r = 10: → прочитай помилку
04
import random — генератор випадкових чисел
randint() · choice() · кожен запуск — новий результат

Модуль random — як кубик всередині Python. Він дає різні числа при кожному запуску, тому програма щоразу малює щось нове.

import random          # ← підключити на початку файлу

x = random.randint(-200, 200)  # ціле число від -200 до 200 включно
y = random.randint(-150, 150)  # кожен запуск → інший результат
r = random.randint(5, 30)      # радіус від 5 до 30

color = random.choice(["red", "blue", "gold"])  # випадковий зі списку
КомандаЩо робить
random.randint(a, b) Повертає випадкове ціле число від a до b включно. Обидва кінці — можливі!
random.choice([...]) Повертає випадковий елемент зі списку
Увага: randint(a, b) — обидва кінці включно!
range(1, 6) дає 1, 2, 3, 4, 5 — без 6. А random.randint(1, 6) може дати 1, 2, 3, 4, 5 або 6 теж. Це класичне місце плутанини між двома функціями. 
# Коло у випадковому місці, випадкового розміру — колір залежить від радіуса
import turtle
import random

t = turtle.Turtle()
t.speed(0)

x = random.randint(-250, 250)   # випадкова позиція X
y = random.randint(-180, 180)   # випадкова позиція Y
r = random.randint(10, 50)      # випадковий радіус

if r > 35:                        # велике коло
    t.pencolor("gold")
elif r > 20:                      # середнє коло
    t.pencolor("red")
else:                              # маленьке коло
    t.pencolor("blue")

t.penup()
t.goto(x, y)
t.pendown()
t.circle(r)

turtle.done()
Спробуй сам!
  • Запусти кілька разів — коло щоразу в іншому місці і іншого розміру?
  • Зміни межі: randint(5, 100) — яке коло може вийти?
  • Використай random.choice(), щоб вибрати колір: pencolor(random.choice(["red","blue","gold"]))
05
for + if — цикл і умова разом
парні і непарні · i % 2 · різні кольори через умову

Справжня сила — коли for і if поєднуються. На кожній ітерації циклу перевіряємо умову і діємо по-різному:

import turtle
t = turtle.Turtle()
t.speed(0)

for i in range(10):
    if i % 2 == 0:         # i = 0, 2, 4, 6, 8 — парні
        t.pencolor("gold")
    else:                   # i = 1, 3, 5, 7, 9 — непарні
        t.pencolor("#5b9cf6")
    t.circle(20)
    t.penup()
    t.forward(50)
    t.pendown()

turtle.done()
Що таке % (модуло)?
i % 2 — остача від ділення i на 2. Парні числа: остача 0 → 4 % 2 == 0. Непарні: остача 1 → 5 % 2 == 1. Саме тому i % 2 == 0 означає «i парне».
for + if (умова від i)
for i in range(6): if i % 2 == 0: t.pencolor("gold") else: t.pencolor("blue") t.circle(20) t.forward(45)
for + if (умова від random)
for i in range(6): r = random.randint(5, 30) if r > 15: t.pencolor("gold") else: t.pencolor("blue") t.circle(r) t.forward(60)
Спробуй сам!
  • Запусти перший варіант — парне золоте, непарне блакитне?
  • Зміни умову на i % 3 == 0 — що змінилось?
  • У другому варіанті зміни межу: r > 15 → r > 20 — більше або менше золотих?
06
Все разом — 20 бульбашок
for + random + if/elif/else · повний приклад · hideturtle

Поєднаємо все, що вивчили сьогодні. Програма малює 20 бульбашок у випадкових місцях, де колір залежить від розміру:

import turtle
import random

t = turtle.Turtle()
t.speed(0)
t.hideturtle()         # прибрати черепашку — малюнок охайніший

for i in range(20):
    x = random.randint(-250, 250)
    y = random.randint(-180, 180)
    r = random.randint(5, 35)

    if r > 25:
        t.pencolor("gold")     # великі → золоті
    elif r > 15:
        t.pencolor("#5b9cf6") # середні → сині
    else:
        t.pencolor("white")   # малі → білі

    t.penup()
    t.goto(x, y)
    t.pendown()
    t.circle(r)

turtle.done()
Нова команда: t.hideturtle()
Ховає значок черепашки (трикутник) з екрана. Малюнок виглядає охайніше. Повернути: t.showturtle().
Спробуй сам!
  • Запусти і натисни ▶ ще раз — кожен запуск інший результат!
  • Збільш range(20) до range(60) — що змінилось?
  • Додай t.fillcolor() + begin_fill() / end_fill(), щоб кола були залиті кольором
  • Зміни межі умов — зроби більше золотих або більше синіх
07
Мала практика — Феєрверк
for · random · if/elif/else · setheading

Намалюй 6 вибухів феєрверку на чорному фоні. Кожен вибух — це промені, що розходяться від випадкової точки в усі сторони. Колір кожного променя залежить від його довжини.

Що має бути в коді:

  • Чорний фон: спочатку screen = turtle.Screen(), потім screen.bgcolor("black")
  • Зовнішній цикл — 6 вибухів, кожен у випадковій точці (x, y)
  • Внутрішній цикл — 24 промені: forward(length) → backward(length) → left(15)
  • Для кожного променя — випадкова довжина від 20 до 80
  • if/elif/else: довгі промені → "gold", середні → "red", короткі → "blue"

  • Бонус: зроби крок range(0, 360, 10) — густіший вибух
