ФУНКЦІЇ DEF

Замість копіювати код — описуємо дію один раз і викликаємо скільки завгодно. def дозволяє створювати власні «команди-цеглинки» і будувати складні малюнки із простих блоків.

Ми вже знаємо ці форми — зробимо їх функціями
урок 3 → функції · квадрат · прямокутник · правильний многокутник · DRY

На уроці 3 ти малював будинок з парканом. Поглянь на те, що ти вже знаєш, під новим кутом — кожна частина сцени складається з кількох простих форм:

ВікноКвадрат або 6-кутник1–2

Стовпчик паркана — це вузький прямокутник. Ти малював його в циклі, щоразу змінюючи x вручну. Якщо загорнути його в функцію — виклик замість 5 рядків стане одним рядком.

Урок 3 — код стовпчика 5 разів
# стовпчик 1 t.penup(); t.goto(-100, -80); t.pendown() t.fillcolor("#8B4513"); t.begin_fill() for s in [10,35,10,35]: t.forward(s); t.right(90) t.end_fill() # стовпчик 2 — той самий блок t.penup(); t.goto(-75, -80); t.pendown() # ... і так 5+ разів
Урок 5 — draw_rect один раз → виклик
def draw_rect(x, y, w, h, color): t.penup(); t.goto(x, y); t.setheading(0) t.fillcolor(color); t.begin_fill() for s in [w, h, w, h]: t.forward(s); t.right(90) t.end_fill() # 5 стовпчиків — 5 рядків for i in range(5): draw_rect(-100 + i*25, -80, 10, 35, "#8B4513")

На цьому уроці ми побудуємо три універсальні функції-цеглинки і складемо з них повну сцену — будинок з парканом, та феєрверк. Ось що будуємо:

01
def — як записати функцію
синтаксис · двокрапка · відступ · спочатку визначити потім викликати

Функція записується з ключовим словом def, потім ім'я, дужки, двокрапка — і тіло з відступом. Ті самі правила, що в for і if:

def назва_функції():   # ← def, ім'я, дужки, двокрапка
    # тіло — виконується при кожному виклику
    t.forward(100)
    t.right(90)

# Виклик — просто ім'я і дужки:
назва_функції()
ЧастинаЩо означає
def Ключове слово — оголошує функцію. Від слова define (визначити)
ім'я_функції Придумуєш сам. Ті самі правила, що для змінних: латиниця, без пробілів, _ замість пробілу
(): Дужки для параметрів (поки порожні) + обов'язкова двокрапка
Відступ у тілі Ті самі правила, що в for і if. Без відступу — IndentationError
виклик() Щоб функція виконалась — її треба викликати. Просто написати ім'я і дужки 
import turtle
t = turtle.Turtle()
t.speed(0)

def draw_square():        # ВИЗНАЧЕННЯ функції
    for _ in range(4):
        t.forward(80)
        t.right(90)

draw_square()             # ВИКЛИК — перший квадрат
t.penup(); t.forward(100); t.pendown()
draw_square()             # ВИКЛИК — другий квадрат

turtle.done()
Порядок має значення!
  • Спочатку визначити функцію (def), потім викликати
  • Якщо викликати до визначення → NameError: name 'draw_square' is not defined
  • Сама функція не виконується при визначенні — тільки при виклику
Спробуй сам!
  • Запусти — два квадрати поруч?
  • Виклич draw_square() ще 3 рази — що виходить?
  • Навмисне перемісти виклик до визначення → прочитай помилку
02
Параметри — гнучкі функції
параметр · аргумент · змінна всередині функції

Функція без параметрів завжди робить одне й те саме. Параметр — це змінна всередині функції, значення якої ми передаємо при виклику:

def draw_square(size):        # size — параметр, жива змінна всередині функції
    for _ in range(4):
        t.forward(size)         # використовуємо size тут
        t.right(90)

draw_square(50)              # size = 50 → квадрат 50px
t.penup(); t.forward(70); t.pendown()
draw_square(120)             # size = 120 → квадрат 120px
Параметр vs Аргумент:
def draw_square(size) — тут size це параметр (ім'я в дужках при визначенні).
draw_square(50) — тут 50 це аргумент (значення при виклику).
На практиці ці слова часто вживають як синоніми — це нормально.

Можна мати кілька параметрів через кому:

def draw_rect(x, y, w, h, color):    # 5 параметрів!
    t.penup()
    t.goto(x, y)
    t.pendown()
    t.setheading(0)              # ← про це — у наступному кроці
    t.fillcolor(color)
    t.begin_fill()
    for side in [w, h, w, h]:
        t.forward(side)
        t.right(90)
    t.end_fill()

draw_rect(-150, 50, 300, 50, "#005BBB")   # синій прямокутник
draw_rect(-150,   0, 300, 50, "#FFD500")   # жовтий прямокутник
Кількість аргументів повинна збігатись з кількістю параметрів!
Якщо функція має 5 параметрів — треба передати рівно 5 аргументів. Більше або менше → TypeError.
Спробуй сам!
  • Запусти draw_rect — два прямокутники один під одним (прапор України!)
  • Зміни параметри: намалюй 3 кольорові смуги одна над одною
  • Навмисне передай 4 аргументи замість 5 → прочитай помилку
03
setheading() — скидаємо напрям
абсолютний кут · 0° вправо · 90° вгору · перший рядок у кожній функції

Черепашка пам'ятає куди вона дивиться після попередніх команд. Якщо функція залишила черепашку повернутою під кутом — наступний виклик тієї ж функції дасть інший результат. setheading() виправляє це:

КомандаНапрям
t.setheading(0)Вправо (→) — стандартний початок
t.setheading(90)Вгору (↑)
t.setheading(180)Вліво (←)
t.setheading(270)Вниз (↓)
Без setheading — другий квадрат кривий
def draw_square(x, y, size): t.penup(); t.goto(x, y); t.pendown() # Не скидаємо напрям! for _ in range(4): t.forward(size); t.right(90) # Перший — нормальний draw_square(-100, 0, 80) t.right(45) # черепашка повернута! # Другий — повернений під кутом 45° draw_square(50, 0, 80)
З setheading — завжди рівний квадрат
def draw_square(x, y, size): t.penup(); t.goto(x, y); t.pendown() t.setheading(0) # ← завжди скидаємо! for _ in range(4): t.forward(size); t.right(90) # Перший — нормальний draw_square(-100, 0, 80) t.right(45) # черепашка повернута # Другий — теж рівний! setheading() виправив draw_square(50, 0, 80)
Правило: перший рядок у кожній функції-малювальнику — setheading(0)
Це робить функцію передбачуваною — незалежно від того, де і як її викликають, вона завжди малює однаково. Це важлива частина написання «чистих» функцій.
Спробуй сам!
  • Запусти обидва варіанти — бачиш різницю?
  • Спробуй setheading(45) — під яким кутом малюється квадрат?
  • Що дає setheading(90) + прямокутник — вертикальна смуга?
04
Три функції-цеглинки → будинок з парканом
draw_rect · draw_triangle · draw_regular_polygon · for у функції · зв'язок з уроком 3

Побудуємо три базові функції. Кожна з них корисна сама по собі, але разом вони дозволяють описати будь-яку сцену кількома рядками.

import turtle
t = turtle.Turtle()
t.speed(0); t.hideturtle()

def draw_rect(x, y, w, h, color):
    # прямокутник — стіна, двері, паркан, будь-що
    t.penup(); t.goto(x, y); t.pendown()
    t.setheading(0)
    t.fillcolor(color); t.begin_fill()
    for side in [w, h, w, h]: t.forward(side); t.right(90)
    t.end_fill()

def draw_triangle(x, y, size, color):
    # рівносторонній трикутник — дах
    t.penup(); t.goto(x, y); t.pendown()
    t.setheading(0)
    t.fillcolor(color); t.begin_fill()
    for _ in range(3): t.forward(size); t.left(120)
    t.end_fill()

def draw_regular_polygon(x, y, n, size, color):
    # правильний многокутник з n сторін — вікно, квітка, зірка
    t.penup(); t.goto(x, y); t.pendown()
    t.setheading(0)
    t.fillcolor(color); t.begin_fill()
    for _ in range(n): t.forward(size); t.left(360 / n)
    t.end_fill()
draw_regular_polygon — одна функція, нескінченно форм:
n=3 → трикутник · n=4 → квадрат · n=5 → п'ятикутник · n=6 → шестикутник · n=12 → майже коло.
360 / n — кут повороту: чим більше сторін, тим менший поворот після кожної.

Тепер збираємо будинок. draw_house викликає інші функції — одна функція всередині іншої. Паркан малюється через for всередині draw_fence:

def draw_house(x, y, size, wall_color, roof_color):
    wall_h = size                                          # висота стіни = size
    draw_rect(x, y, size, wall_h, wall_color)              # стіна
    win = size // 4                                        # розмір вікна — похідний від size
    draw_regular_polygon(x + size//4, y + size//2, 4, win, "#87CEEB")  # вікно-квадрат
    door_w = size // 3
    draw_rect(x + size//3, y, door_w, size//2, "#8B4513")  # двері
    draw_triangle(x, y + wall_h, size, roof_color)         # дах зверху

def draw_fence(x, y, width, color):
    post_w = 8
    step   = 20                                            # крок між стовпчиками
    for i in range(width // step):                         # скільки стовпчиків влізе
        draw_rect(x + i * step, y, post_w, 28, color)
    draw_rect(x, y + 18, width, 6, color)                  # горизонтальна планка

# ── Будуємо сцену ──
# Трава
draw_rect(-280, -120, 560, 60, "#4a7c2f")

# Два будинки з парканами
draw_house(-220, -60, 90, "#f0c87a", "#c0392b")
draw_fence(-240, -120, 130, "#8B4513")

draw_house(  30, -60, 90, "#a8d8ea", "#6c3483")
draw_fence(  10, -120, 130, "#5d4037")

turtle.done()
Змінні всередині функції: win = size // 4, door_w = size // 3 — ці значення автоматично масштабуються разом із size. Зміниш один параметр — весь будинок перемасштабується пропорційно.
Спробуй сам!
  • Запусти — два будинки, трава, два паркани?
  • Зміни size=90 на size=60 у першому будинку — всі частини стали меншими?
  • Заміни n=4 у вікні на n=6 — шестикутне вікно замість квадратного
  • Додай третій будинок з x=200 і парканом поруч
  • Бонус: зроби for i in range(3): для вулиці з трьох будинків через списки кольорів
05
Рефакторинг — Феєрверк через функції
зв'язок з уроком 4 · draw_explosion() · той самий результат — чистіший код

На уроці 4 ти малював феєрверк. Погляньмо на той код і переробимо його через def — без зміни результату, але з набагато зрозумілішою структурою.

Урок 4 — без функцій
for i in range(6): x = random.randint(-220, 220) y = random.randint(-150, 150) t.penup(); t.goto(x, y) t.setheading(0) for angle in range(0, 360, 15): length = random.randint(20, 80) if length > 60: t.pencolor("gold") elif length > 40: t.pencolor("red") else: t.pencolor("blue") t.pendown() t.forward(length) t.backward(length) t.left(15)
Урок 5 — з функцією
def draw_explosion(x, y): t.penup(); t.goto(x, y) t.setheading(0) for angle in range(0, 360, 15): length = random.randint(20, 80) if length > 60: t.pencolor("gold") elif length > 40: t.pencolor("red") else: t.pencolor("blue") t.pendown() t.forward(length) t.backward(length) t.left(15) for i in range(6): x = random.randint(-220, 220) y = random.randint(-150, 150) draw_explosion(x, y)

Результат на екрані — абсолютно однаковий. Але тепер зовнішній цикл читається як текст: «6 разів — намалюй вибух у випадковому місці». Логіка вибуху прихована всередині функції і не засмічує основний код.

Що змінилось і чому це краще:
Хочеш намалювати 12 вибухів? Міняєш тільки range(6) → range(12).
Хочеш густіший вибух? Міняєш тільки крок у range(0, 360, 15) — всередині функції.
Хочеш ще одну позицію вибуху? draw_explosion(0, 0) — один рядок.

Ось повний код для запуску:

import turtle
import random

screen = turtle.Screen()
screen.bgcolor("black")

t = turtle.Turtle()
t.speed(0)
t.hideturtle()

def draw_explosion(x, y):
    t.penup()
    t.goto(x, y)
    t.setheading(0)

    for angle in range(0, 360, 15):
        length = random.randint(20, 80)
        if length > 60:
            t.pencolor("gold")
        elif length > 40:
            t.pencolor("red")
        else:
            t.pencolor("blue")
        t.pendown()
        t.forward(length)
        t.backward(length)
        t.penup()
        t.left(15)

for i in range(6):
    x = random.randint(-220, 220)
    y = random.randint(-150, 150)
    draw_explosion(x, y)

turtle.done()
Спробуй сам!
  • Запусти — той самий феєрверк, що на уроці 4, але тепер через функцію
  • Зміни range(6) на range(12) — стало більше вибухів без змін у функції?
  • Зміни крок на range(0, 360, 10) — густіший вибух?
  • Бонус: додай параметр draw_explosion(x, y, step=15) і виклич з різними кроками для різних вибухів
06
Практика — Квіти на галявині
draw_regular_polygon · for · random · змінна з параметра · пелюстки навколо центра

Намалюй галявину з 8–10 квіток. Кожна квітка — це 6 пелюсток і жовтий центр. Пелюстки — це кола, розставлені по колу. Це перше самостійне завдання уроку — пробуй, роби помилки, читай підказки.

Як влаштована одна квітка: 6 пелюсток рівномірно навколо центра. Кожна пелюстка — це коло. Щоб розмістити коло навколо центра, черепашка дивиться у бік пелюстки (setheading), рухається вперед на відстань size, малює коло радіусом size // 2, потім повертається в центр (goto).

Що треба зробитиЯким інструментом
Намалювати одну квітку def draw_flower(x, y, size, color):
Розмір пелюстки залежить від size Змінна всередині: petal = size // 2
6 пелюсток рівномірно for angle in range(0, 360, 60): + setheading(angle)
Поставити черепашку до пелюстки forward(size) → малюємо коло → penup(); goto(x, y)
Жовтий центр поверх пелюсток goto(x, y) → circle(petal // 2) із жовтим fill
Зелений фон draw_rect(-280, -160, 560, 80, "#2d5a1b") — трава знизу
8–10 квіток у випадкових місцях Зовнішній for + random.randint() + random.choice()
Типова пастка — черепашка не повертається в центр:
Після forward(size) черепашка знаходиться біля пелюстки. Після малювання кола — там само. Щоб намалювати наступну пелюстку від центра, потрібно зробити penup(); goto(x, y) явно — backward(size) не спрацює, бо напрям після кола змінився.
Здай роботу (1-й рівень)
  • Функція draw_flower написана і використана
  • 8+ квіток у випадкових місцях, зелений фон-трава
  • Змінна petal = size // 2 є всередині функції
  • Бонус: random.choice(["#ff9ff3", "#ff6b6b", "#feca57", "#48dbfb"]) для кольору
  • Бонус 2: якщо квітка велика (size > 25) — if/elif для зміни кольору центра
07
Практика — Поле сніжинок
def · параметри · змінна з параметра · for · random · все разом

Намалюй поле з 15 сніжинок на чорному фоні. Кожна сніжинка — у випадковому місці, випадкового розміру. Завдання вимагає поєднати всі чотири інструменти уроку: def, параметри, for і random.

Як влаштована одна сніжинка: 6 гілок, рівномірно через кожні 60°. На кожній гілці — дві коротші гілочки, відхилені на 60° в боки. Довжина гілочок залежить від розміру сніжинки.

Що треба зробитиЯким інструментом
Намалювати одну сніжинку def draw_snowflake(x, y, size): — функція з трьома параметрами
Розрахувати довжину гілочок Змінна всередині функції: branch = size // 3
6 гілок рівно через 60° for angle in range(0, 360, 60): + setheading(angle)
Намалювати гілку і повернутись forward(size) → дві гілочки → backward(size)
15 сніжинок у випадкових місцях Зовнішній for + random.randint() для x, y, size
Підказка — як намалювати одну гілку з гілочками:
Встанови напрям → forward(size) → повернись на 60° ліворуч → forward(branch) → backward(branch) → повернись на 120° праворуч → forward(branch) → backward(branch) → повернись на 60° ліворуч (назад до напряму гілки) → backward(size).
Не забудь:
  • На початку функції: setheading(0) — інакше сніжинки будуть повернуті
  • penup() перед goto(), pendown() перед малюванням
  • Чорний фон: screen = turtle.Screen() → screen.bgcolor("black")
